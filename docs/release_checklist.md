# 먹통 방지 체크리스트

앱 수정·배포 전 아래 항목을 점검하면 무한 루프나 화면 멈춤 같은 치명적 이슈를 예방할 수 있습니다.

1. **반복문/타이머 상한 확인**  
   - 난수 배치, 재귀 호출 등에는 항상 최대 반복 횟수 또는 타임아웃이 있는지 점검합니다.  
   - 실패 시 간격 완화·폴백 루프가 있는지 체크하고, 로그로 감지 가능하게 만듭니다.

2. **비동기 콜백 가드**  
   - `Future.delayed`, `Timer`, 애니메이션 콜백 진입 시 `if (!mounted) return;` 등 안전 장치를 넣습니다.  
   - 화면 전환(Back, Navigator.pop 등) 전에 타이머·컨트롤러를 `cancel`/`dispose` 해 중복 호출을 막습니다.

3. **상태 업데이트 방식 점검**  
   - UI에 영향을 주는 값은 반드시 `setState` 안에서 갱신하고, 예외 시 사용자에게 안내하거나 기본 상태로 되돌립니다.

4. **고밀도 시나리오 테스트**  
   - Stage 20 이상, 구름 40개 이상 등 극단 상황을 수동/자동 테스트해 배치 로직이 끝까지 종료되는지 확인합니다.  
   - 노드 수나 최소 간격을 변경했다면 관련 테스트도 업데이트합니다.

5. **리소스 누수 확인**  
   - 오디오 플레이어, 애니메이션 컨트롤러 등은 `dispose`에서 해제하고, 중간에 예외 로그가 없는지 확인합니다.

6. **자동화 + QA 루틴 유지**  
   - `flutter test` 외에 위험이 큰 기능(배치, 타이머, 오디오 재생)은 개별 유닛 테스트를 추가/유지합니다.  
   - QA 체크리스트: 고스테이지 진행, 빠른 스와이프 연속 입력, 승/패 반복, 앱 백그라운드↔복귀 시 동작 등을 포함합니다.

이 문서를 최신 상태로 유지하면서, 새로운 위험 요소가 보이면 항목을 추가하거나 수정해 주세요.
